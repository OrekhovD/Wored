# WORED Project Context & Handoff (Security & UI/UX v2)

**Дата:** 01.05.2026
**Статус:** Базовая безопасность и Command Deck v2 успешно интегрированы.

## 1. Архитектура и Безопасность (Security Hardening)
Мы провели аудит безопасности и изолировали критические уязвимости:
- **База данных:** PostgreSQL теперь использует изолированный файл `.env.postgres` (добавлен в `.gitignore`), что исключает утечку API-ключей бирж (`EXCHANGE_API_KEY`, `EXCHANGE_SECRET_KEY`) в среду базы данных.
- **Предотвращение утечек:** Добавлен git pre-commit hook (`scripts/pre-commit-check-env.sh` и `scripts/install-hooks.ps1`), блокирующий случайный коммит любых `.env` файлов.
- **Генерация секретов:** Написан кроссплатформенный скрипт `scripts/generate_secrets.py` для безопасной генерации пароля БД и `WEBUI_SESSION_SECRET`.
- **Инициализация:** В `Makefile` добавлена цель `setup`, автоматизирующая первичную настройку.

**Важно:** Существующие API-ключи бирж оставлены **без ротации** (строгое требование пользователя).

## 2. WebUI: Command Deck v2 (UI/UX)
Интерфейс переведен в формат операционного терминала (Command Deck) с использованием неразрушающего (аддитивного) патча поверх существующей архитектуры (`app.js` и 6 страниц):

### Ключевые изменения:
1. **UX Foundation:** В `styles.css` добавлены семантические токены (`--status-ok`, `--status-warn`, `--status-bad`, `--surface-1/2/3`) и классы состояний (`empty-state`, `loading-skeleton`).
2. **Topbar Health & Alerts:** В `base.html` добавлена глобальная полоса здоровья (`topbar-health`) для Redis, PG и Collector, а также счетчик алертов. Данные обновляются фоновым JS-скриптом без перезагрузки страниц (fetch к `/api/health` и `/api/alerts?limit=1`).
3. **Market Focus Cockpit:** На странице Dashboard (`index.html`) заменен декоративный Hero-блок. Теперь там выводится Cockpit с живыми данными: цена, дельта, RSI, MACD, тренд SMA.
4. **Operational Pulse:** Панель быстрого мониторинга инфраструктуры на Dashboard.
5. **Alerts Triage:** В `alerts.html` добавлена классификация по Severity (Critical, High, Medium, Low) на основе порогового значения (`threshold`), с цветовой индикацией границ строк.
6. **AI Journal Inspector:** Сырые JSON-данные в `journal.html` теперь скрыты под сворачиваемыми блоками (`collapsible-toggle`), улучшая читаемость.
7. **Prediction Lab:** Добавлен расчет разрыва (`Gap`) между предсказанным и фактическим процентом изменения. Оформлены Empty States.
