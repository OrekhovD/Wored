# Runtime Snapshot Playbook

## Что это?
`runtime_snapshot.sh` — это скрипт, который собирает безопасный срез текущего состояния проекта WORED.
Он извлекает:
- Логи контейнеров (ошибки)
- Доступность WebUI маршрутов
- Содержимое основных таблиц Postgres (`alerts`, `forecast_requests`, `ai_journal`)
- Статистику Redis
- Статус репозитория Git

## Для чего это нужно?
Когда ты (как агент Hermes) сталкиваешься с задачей диагностики багов, обрывов сети или странного поведения WebUI, ты можешь одной командой `/snapshot` собрать всё необходимое. Это избавляет от необходимости вручную дёргать `docker compose ps`, проверять каждую базу и парсить логи.

## Безопасность
- **Секретов нет:** Все пароли и токены вырезаются через регулярные выражения.
- **Read-Only:** Скрипт использует только команды `GET`, `ping`, `SELECT` и `logs`. Никаких `DROP`, `DELETE` или `UPDATE`.
- **Без обрушения системы:** Если Postgres недоступен или пуст, скрипт не упадет, а просто зафиксирует этот факт.

## Как использовать
Введи `/snapshot` в терминале Hermes.
В ответ ты получишь сообщение, где лежат новые логи (внутри директории `runtime_snapshot/YYYYMMDD_HHMMSS/`). Загляни в эти txt-файлы для подробного анализа проблемы.
