# Безопасность WORED

## Модель угроз

WORED — это локальный бот. Угрозы, от которых он защищён в текущей архитектуре:
1. **Утечка ключей в репозиторий**: Исключено за счёт `.gitignore` и pre-commit hook.
2. **Перехват ключей соседними контейнерами**: Контейнер Postgres изолирован от ключей AI-моделей (использует отдельный `.env.postgres`).
3. **Несанкционированный доступ к WebUI**: Закрыт за счёт `WEBUI_AUTH_ENABLED=true` по умолчанию и сильных сгенерированных паролей.
4. **Утечка токенов биржи**: HTX API ключи нужны только для read-only, бот не умеет выполнять торговые операции.

## Контроль окружений

В проекте используются два независимых окружения:
1. **Основное (WORED)**: конфигурация в корне проекта (`.env`).
2. **Gateway (Hypercube)**: конфигурация в папке `hypercube/` (`hypercube/.env`).

Конфигурация не смешивается. Если вы используете Hypercube, вы должны отдельно настроить его `.env` на основе его `.env.example`.

## Управление секретами

Никогда не используйте одинаковые пароли и пустые секреты.

### Генерация секретов

Для генерации надёжных криптографически стойких паролей используйте встроенный скрипт:

```powershell
python scripts/generate_secrets.py
```

Скрипт автоматически:
1. Сгенерирует `POSTGRES_PASSWORD`, `WEBUI_ADMIN_PASSWORD` и `WEBUI_SESSION_SECRET`.
2. Создаст файл `.env.postgres` для изоляции базы данных.
3. Выведет значения на экран, чтобы вы скопировали их в основной `.env` файл.

### Ротация и обновление ключей

В рамках текущей версии WORED **автоматическая ротация ключей не предусмотрена и не требуется**.
Если вы подозреваете компрометацию (например, случайно отправили ключ в публичный чат), вы можете обновить их вручную:

1. Запросите новые API-ключи (DashScope, Telegram, и т.д.).
2. Остановите стек: `docker-compose down`.
3. Откройте `.env` и замените скомпрометированные значения.
4. Если меняете `POSTGRES_PASSWORD`, вам нужно будет удалить volume `postgres_data` (внимание: удалит историю!). Если история важна, меняйте пароль пользователя базы данных внутри контейнера.
5. Запустите стек снова: `docker-compose up -d`.

## Защита от случайного коммита

Чтобы избежать утечки файла `.env` в публичный репозиторий:

1. Убедитесь, что вы запустили `scripts\install-hooks.ps1`.
2. Этот скрипт установит локальный pre-commit hook.
3. Hook будет блокировать любой `git commit`, если в индексе находится `.env` или `.env.postgres`.
