\#\# 3\. КОМПОНЕНТЫ СИСТЕМЫ

\#\#\# 3.1. Collector — основные задачи

\*\*WebSocket стрим (HTX):\*\*  
\- Endpoint: \`wss://api.huobi.pro/ws\`  
\- Подписки на символы из WATCHLIST  
\- Каналы:  
  \- \`market.{symbol}.detail\` — тикеры (10/sec)  
  \- \`market.{symbol}.kline.1min\` — свечи 1м  
  \- \`market.{symbol}.kline.5min\` — свечи 5м  
  \- \`market.{symbol}.depth.step0\` — стакан (топ 20\)

\*\*Важно:\*\* HTX возвращает GZIP-сжатые данные\!

\*\*REST API (досинхронизация):\*\*  
\- Endpoint: \`https://api.huobi.pro\`  
\- \`/market/history/kline\` — исторические свечи  
\- Загрузка при старте: 300 свечей на каждый таймфрейм  
\- Периодическая синхронизация: каждые 5 минут (1ч, 1д)

\*\*Индикаторы (pandas-ta):\*\*  
\`\`\`python  
RSI(14) → перекуплен \>70, перепродан \<30  
MACD(12,26,9) → бычье/медвежье пересечение  
Bollinger Bands(20,2) → позиция цены в полосах  
EMA(20,50) → краткосрочный тренд  
ATR(14) → волатильность  
\`\`\`

\*\*Автоалерты (detector.py):\*\*  
1\. \*\*Спайки цены\*\* — изменение ≥3% за 15 минут  
2\. \*\*RSI экстремумы\*\* — RSI ≥75 или ≤25  
3\. \*\*MACD cross\*\* — пересечение сигнальной линии  
4\. \*\*BB break\*\* — выход за верхнюю/нижнюю полосу  
5\. \*\*Аномальный объём\*\* — ≥3x среднего

\*\*Cooldown алертов:\*\*  
\- SPIKE: 5 минут  
\- RSI: 10 минут  
\- MACD: 5 минут  
\- BB: 3 минуты

\#\#\# 3.2. Scheduler — фоновые задачи

\*\*Alert Checker (alert\_checker.py):\*\*  
\- Запуск: каждые 30 секунд  
\- Задача: проверка пользовательских ценовых алертов  
\- Логика:  
  1\. SELECT все active алерты из PostgreSQL  
  2\. GET текущая цена из Redis  
  3\. Сравнение: если условие сработало → Telegram уведомление  
  4\. UPDATE алерт: active=FALSE, triggered\_at=NOW()

\*\*Briefing (briefing.py):\*\*  
\- Время: 08:00 МСК ежедневно \+ 20:00 МСК воскресенье  
\- Задача: утренняя аналитика и прогноз  
\- Логика:  
  1\. Сбор снапшота рынка (цены, индикаторы, алерты за ночь)  
  2\. Запрос к GLM-5.1: прогноз на день по часам (10/12/14/16/18/20 МСК)  
  3\. Рассылка всем активным пользователям (за 48ч)

\#\#\# 3.3. Journal — AI-контекст

\*\*Realtime снапшот (30 сек):\*\*  
\`\`\`json  
{  
  "timestamp": "2026-04-18T05:00:00Z",  
  "snapshot": {  
    "btcusdt": {"price": 84320.5, "change\_24h": "+1.2%", ...},  
    "ethusdt": {...}  
  },  
  "indicators": {  
    "btcusdt": {"rsi": 62.1, "macd\_signal": "bullish\_cross", ...}  
  },  
  "alerts": \[...\],  
  "market\_summary": "Общий сентимент: бычий..."  
}  
\`\`\`  
\- Хранение: Redis ключ \`ai:journal:latest\` (TTL 120 сек)

\*\*Hourly снапшот (1 час):\*\*  
\- Сохранение в PostgreSQL таблица \`ai\_journal\`  
\- Тип: \`hourly\`  
\- Для ретроспективного анализа

\#\#\# 3.4. Chatbot — handlers

\*\*start.py:\*\*  
\- \`/start\` — главное меню (адаптивное)  
\- \`/help\` — справка  
\- Онбординг для новых пользователей (5 шагов)

\*\*market.py:\*\*  
\- \`/price \[symbol\]\` — карточка цены с индикаторами  
\- \`/analysis \[symbol\]\` — выбор таймфрейма → AI-анализ  
\- \`/indicators\` — сводка по всем парам  
\- \`/position\` — FSM расчёт позиции (депозит → риск → уровни)

\*\*chat.py:\*\*  
\- Свободный текст → роутер → AI-модель  
\- История чата в PostgreSQL  
\- Контекстные кнопки после ответа

\*\*alerts.py:\*\*  
\- \`/alerts\` — список активных алертов  
\- FSM создание алерта: символ → цена → направление  
\- Удаление всех алертов

\*\*portfolio.py:\*\*  
\- \`/portfolio\` — открытые позиции \+ реалтайм PnL  
\- FSM открытие позиции: символ → направление → вход → объём → SL/TP  
\- Закрытие позиции с подтверждением  
\- История закрытых позиций

\*\*analytics.py:\*\*  
\- Ретроспектива: выбор пары → период → GLM-5.1 анализ  
\- Бэктест: 5 стратегий \+ кастомная → QwenCode генерация кода  
\- Экспорт CSV: пара → интервал → период → файл в чат  
\- Статистика алертов (24ч/7д)  
\- Топ движений дня

\*\*settings.py:\*\*  
\- Вотчлист (до 10 пар)  
\- Модель по умолчанию  
\- Брифинг вкл/выкл

\#\#\# 3.5. AI Router (из router.py)

\*\*Intent classification:\*\*  
\`\`\`python  
market\_news     → Perplexity Sonar Pro  
deep\_analysis   → GLM-5.1 (thinking mode)  
backtest\_code   → QwenCode3-Plus  
position\_calc   → Qwen3.6-Plus  
quick\_chat      → MiniMax M2.7  
alert\_setup     → MiniMax M2.7  
\`\`\`

\*\*Fallback цепочка:\*\*  
\- Если модель недоступна → следующая в цепочке  
\- Если все AI не работают → Qwen3.6-Plus (гарантированная квота)

\#\# 6\. API И ИНТЕГРАЦИИ

\#\#\# 6.1. HTX WebSocket API

\*\*Endpoint:\*\* \`wss://api.huobi.pro/ws\`

\*\*Подписка:\*\*  
\`\`\`python  
await ws.send(json.dumps({  
    "sub": f"market.{symbol}.detail",  
    "id": f"detail\_{symbol}"  
}))  
\`\`\`

\*\*Ping/Pong:\*\*  
\`\`\`python  
if "ping" in msg:  
    await ws.send(json.dumps({"pong": msg\["ping"\]}))  
\`\`\`

\*\*GZIP decompression:\*\*  
\`\`\`python  
import gzip  
msg \= json.loads(gzip.decompress(raw\_msg).decode("utf-8"))  
\`\`\`

\#\#\# 6.2. HTX REST API

\*\*Endpoint:\*\* \`https://api.huobi.pro\`

\*\*Получение свечей:\*\*  
\`\`\`python  
GET /market/history/kline  
Params: symbol=btcusdt, period=1min, size=300  
\`\`\`

\*\*Rate limit:\*\* 100 запросов/10 секунд

\#\#\# 6.3. AI Models API

\*\*Qwen (DashScope):\*\*  
\- Endpoint: \`https://dashscope.aliyuncs.com/compatible-mode/v1\`  
\- Model: \`qwen3.6-plus\` (основная)  
\- Auth: \`DASHSCOPE\_API\_KEY\`

\*\*GLM (ZhipuAI):\*\*  
\- Endpoint: \`https://open.bigmodel.cn/api/paas/v4/\`  
\- Model: \`glm-4-plus\` (уточнить GLM-5.1)  
\- Auth: \`GLM\_API\_KEY\`

\*\*MiniMax (NVIDIA NIM):\*\*  
\- Endpoint: \`https://integrate.api.nvidia.com/v1\`

\#\# 📝 ПРИМЕЧАНИЯ

\#\#\# Важные моменты

1\. \*\*HTX WebSocket GZIP:\*\*  
   \- Обязательно \`gzip.decompress()\` перед \`json.loads()\`  
   \- Некоторые сообщения не сжаты (ping) — try/except

2\. \*\*Reconnect логика:\*\*  
   \- Delays: \[1, 2, 4, 8, 16, 30\] секунд  
   \- Сброс счётчика после успешного подключения

3\. \*\*AI rate limits:\*\*  
   \- Qwen: 505k/1M квота (бесплатно до 04.06.2026)  
   \- GLM: уточнить limits  
   \- MiniMax: через NVIDIA NIM (другие limits)

4\. \*\*Telegram rate limits:\*\*  
   \- \~30 сообщений/сек глобально  
   \- Использовать \`asyncio.sleep(0.1)\` в broadcast

5\. \*\*PostgreSQL connection pool:\*\*  
   \- Collector: min\_size=2, max\_size=10  
   \- Chatbot: min\_size=2, max\_size=5

\#\#\# Известные ограничения

1\. \*\*GLM-5.1 model string:\*\*  
   \- Требуется проверка в консоли ZhipuAI  
   \- Возможно \`glm-4-plus\` вместо \`glm-5.1\`

2\. \*\*MiniMax через NVIDIA:\*\*  
   \- Точный model name неизвестен  
   \- Требуется проверка в https://build.nvidia.com

3\. \*\*Qwen Coder:\*\*  
   \- \`qwen-coder-3-plus\` может быть недоступен  
   \- Fallback на \`qwen2.5-coder-32b-instruct\`
