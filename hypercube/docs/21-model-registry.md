# Model Registry

## 1. Overview

ModelRegistry отвечает за загрузку и управление метаданными моделей. Модели загружаются из конфигурационного файла, с падением на жестко заданные значения по умолчанию. Регистр поддерживает обновления статуса моделей и предоставляет информацию о доступности моделей и провайдеров.

## 2. Функциональность

- **Загрузка моделей:** Загружает модели из конфигурационного файла или использует жестко заданные значения по умолчанию.
- **Обновление статуса:** Обновляет статусы моделей.
- **Проверки доступности:** Проверяет доступность моделей и провайдеров.
- **Кэширование:** Кэширует загруженные данные для повторного использования без перезагрузки.

## 3. Методы

### 3.1 `_load_from_config`
**Описание:**
- Загружает модели из конфигурационного файла.
- Использует кэш, если уже загружен.
- Если файл недоступен, использует жестко заданные значения по умолчанию.

### 3.2 `is_config_driven`
**Возвращает:**
- `bool` — True, если модели загружены из конфигурационного файла, иначе False.

### 3.3 `reload`
**Описание:**
- Перезагружает модели из конфигурационного файла.
- Очищает текущие данные и загружает новые.

### 3.4 `get_model`
**Параметры:**
- `model_id`: str — ID модели.

**Возвращает:**
- `dict | None` — информация о модели или None, если модель не найдена.

### 3.5 `get_active_models`
**Параметры:**
- `provider_id`: Optional[str] — ID провайдера (опционально).

**Возвращает:**
- `List[dict]` — список активных моделей для данного провайдера или всех провайдеров, если `provider_id` не указан.

### 3.6 `get_models_by_provider`
**Параметры:**
- `provider_id`: str — ID провайдера.

**Возвращает:**
- `List[dict]` — список моделей для данного провайдера.

### 3.7 `get_all_models`
**Возвращает:**
- `List[dict]` — список всех моделей.

### 3.8 `update_model_status`
**Параметры:**
- `model_id`: str — ID модели.
- `status`: str — новый статус модели.

### 3.9 `is_model_available`
**Параметры:**
- `model_id`: str — ID модели.
- `mode`: RoutingMode | str — режим маршрутизации (free_only, balanced, premium).

**Возвращает:**
- `bool` — True, если модель доступна для данного режима, иначе False.

### 3.10 `get_provider_ids`
**Возвращает:**
- `List[str]` — список ID всех провайдеров.

### 3.11 `is_provider_enabled`
**Параметры:**
- `provider_id`: str — ID провайдера.

**Возвращает:**
- `bool` — True, если провайдер включен, иначе False.

### 3.12 `get_provider_config`
**Параметры:**
- `provider_id`: str — ID провайдера.

**Возвращает:**
- `Dict[str, Any]` — полная конфигурация провайдера.

## 4. Принцип работы

### 4.1 Загрузка моделей
ModelRegistry использует объект `_load_registry_from_yaml` для загрузки моделей из конфигурационного файла. Если файл недоступен, используются жестко заданные значения по умолчанию. Загруженные данные кэшируются для повторного использования без перезагрузки.

### 4.2 Обновление статуса
ModelRegistry позволяет обновлять статусы моделей, что полезно для управления доступностью и состоянием моделей.

### 4.3 Проверки доступности
ModelRegistry проверяет доступность моделей и провайдеров, что необходимо для корректной маршрутизации запросов.

### 4.4 Кэширование
ModelRegistry кэширует загруженные данные, чтобы избежать повторной загрузки при последующих вызовах.

## 5. Пример использования

### 5.1 Создание ModelRegistry
```python
from routing.model_registry import ModelRegistry

model_registry = ModelRegistry()
```

### 5.2 Получение модели
```python
model = model_registry.get_model("glm-5")
print(model)
```

### 5.3 Получение активных моделей
```python
active_models = model_registry.get_active_models(provider_id="zhipu")
print(active_models)
```

### 5.4 Обновление статуса модели
```python
model_registry.update_model_status("glm-5", status="degraded")
```

### 5.5 Проверка доступности модели
```python
is_available = model_registry.is_model_available("glm-5", mode=RoutingMode.PREMIUM)
print(is_available)
```

### 5.6 Получение ID провайдеров
```python
provider_ids = model_registry.get_provider_ids()
print(provider_ids)
```

### 5.7 Проверка включения провайдера
```python
is_enabled = model_registry.is_provider_enabled("zhipu")
print(is_enabled)
```

### 5.8 Получение конфигурации провайдера
```python
provider_config = model_registry.get_provider_config("zhipu")
print(provider_config)
```

## 6. Файлы и зависимости

| Файл | Назначение |
|------|-----------|
| `model_registry.py` | Основной класс ModelRegistry |
| `policies.py` | Политики маршрутизации |
| `provider_registry.yaml` | Конфигурационный файл провайдеров |

ModelRegistry зависит от `policies.py` для загрузки конфигурационного файла и получения информации о провайдерах.
