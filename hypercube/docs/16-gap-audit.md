# Gap Audit Report

## ID: SEC-001
**Название:** Утечка чувствительных данных (ключи API) в репозитории
**Категория:** security
**Severity:** blocker
**Текущее состояние:** В репозитории содержатся файлы с реальными ключами API: `.env`, `docs/kluch.txt`, `settings.json`
**Проблема:** Хранение чувствительных данных в открытом виде в репозитории создает серьезную уязвимость
**Требуется исправить:** 
- Удалить чувствительные данные из файлов
- Добавить файлы в .gitignore
- Заменить ключи на placeholder-значения в примерах
**Как проверить исправление:** grep не должен находить реальные ключи в отслеживаемых файлах

## ID: ARCH-001
**Название:** Нарушение архитектурной структуры в модуле core
**Категория:** architecture
**Severity:** high
**Текущее состояние:** В директории core/ находятся файлы, которые не относятся к этому модулю: db.py, engine.py, logger.py, а также поддиректории context/ и providers/
**Проблема:** Нарушение принципа модульности и разделения ответственности
**Требуется исправить:** 
- Переместить db.py и engine.py в storage/
- Переместить logger.py в общую директорию или удалить если не используется
- Переместить поддиректории context/ и providers/ в корень проекта
**Как проверить исправление:** Директория core/ должна содержать только общие схемы, конфигурацию, перечисления и исключения

## ID: TEST-001

**Категория:** testing
**Severity:** high

**Проблема:** Отсутствие unit-тестов не позволяет гарантировать корректность работы отдельных компонентов

## ID: QUOTAS-001
**Название:** Недостаточная проверка квот в обработчике команд
**Категория:** quotas
**Severity:** high
**Текущее состояние:** В обработчике команды /ask нет проверки квот перед отправкой запроса
**Проблема:** Пользователь может отправить запрос, который превысит квоту
**Требуется исправить:** 
- Добавить проверку квоты перед выполнением запроса к AI
- Реализовать предупреждение пользователя при низком остатке
**Как проверить исправление:** При низкой квоте пользователь получает предупреждение до отправки запроса
