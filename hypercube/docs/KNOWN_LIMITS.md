# Known Limits

Известные ограничения и технические долги проекта Hypercube (Telegram AI Gateway для HTX).

## 🔴 Критические ограничения

### 1. Тестирование

| ID | Описание | Влияние | Workaround | Приоритет |
|----|----------|---------|------------|-----------|
| TEST-001 | Нет тестов для Telegram бота | Невозможность автоматической проверки handlers | Ручное тестирование через Telegram | Высокий |
| TEST-002 | Нет mock провайдеров для интеграционных тестов | Тесты требуют реальных API ключей | Использовать test режим с отключенными вызовами | Высокий |
| TEST-003 | Нет тестов для HTX adapter | Риск регрессии при изменении API HTX | Ручная проверка рыночных данных | Средний |
| TEST-004 | Нет тестов для accounting/quotas | Риск ошибок в учете токенов | Мониторинг логов accounting | Средний |

### 2. Интеграции

| ID | Описание | Влияние | Workaround | Приоритет |
|----|----------|---------|------------|-----------|
| INT-001 | HTX API только read-only | Нет торговых операций | Использовать для аналитики | Низкий |
| INT-002 | Нет WebSocket для HTX | Данные не в реальном времени | Polling каждые N секунд | Средний |
| INT-003 | Ограниченное число провайдеров | Зависимость от доступности | Fallback на резервные модели | Высокий |

### 3. Производительность

| ID | Описание | Влияние | Workaround | Приоритет |
|----|----------|---------|------------|-----------|
| PERF-001 | Нет кэширования ответов | Повторные запросы к провайдерам | Добавить Redis кэш | Средний |
| PERF-002 | Нет rate limiting на уровне бота | Риск блокировки Telegram | Очереди запросов | Высокий |
| PERF-003 | Синхронные вызовы БД | Блокировка при большой нагрузке | Перейти на asyncpg | Низкий |
